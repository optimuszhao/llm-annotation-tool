# LLM 标注工具功能说明书

版本基准：当前代码仓库主工程  
适用范围：本地运行版 `FastAPI + SQLite + HTML/JS` 标注平台  
默认入口：`http://127.0.0.1:8000`

---

## 1. 文档目的

这份文档用于说明当前系统已经具备的能力，包括每个功能的作用、可完成的操作、前端交互方式、后端处理逻辑和使用注意事项。

阅读对象：

- 业务使用人员
- 标注执行人员
- 负责接入大模型和业务方法的开发人员
- 维护本地部署环境的实施人员

---

## 2. 系统概览

当前系统围绕三个核心页面工作：

1. `数据集与方案管理`
2. `标注工作台`
3. `对话大模型`

系统主目标：

- 管理不同场景下的数据集和资源
- 组合 Prompt、知识库、错题集与字段映射形成标注方案
- 对数据集执行单条、批量、全量标注
- 实时查看标注状态、指标和结果
- 对单行数据进行分析、收藏、编辑和历史回看

## 5. 数据组织模型

系统的组织层级如下：

```text
场景
  -> 数据集
  -> Prompt
  -> 知识库
  -> 错题集
  -> 字段映射配置
  -> 标注方案
```

### 5.1 场景

作用：

- 作为资源隔离单元
- 不同场景下的数据、Prompt、知识库、错题集、字段映射、方案独立管理

特点：

- 平铺结构
- 每个场景有唯一 `scene_id`
- 每个场景对应独立数据表

### 5.2 数据集

作用：

- 承载 Excel 导入后的标注原始数据
- 为工作台表格提供数据源

特点：

- 支持单文件导入
- 支持多文件导入
- 导入后写入 SQLite
- 前端列表按页加载，不一次加载全部

### 5.3 Prompt

作用：

- 作为标注输入模板
- 可被方案引用

特点：

- 支持名称、角色名、正文内容
- 支持多个 Prompt 同时组成一个方案

### 5.4 知识库

作用：

- 为 Prompt 提供规则、背景、业务说明

### 5.5 错题集

作用：

- 为 Prompt 提供错误案例参考
- 当前阶段以资源管理为主

### 5.6 字段映射配置

作用：

- 告诉系统哪些列参与判定、渲染和标注

核心配置：

- 人工答案列
- 标注答案列
- 默认渲染列
- 标注使用列

### 5.7 标注方案

作用：

- 把 Prompt、标注方法、Prompt 初始化方式、并发参数组织成可执行方案

## 7. 场景管理

### 7.1 功能作用

场景用于隔离不同业务线、不同测试任务或不同项目的数据和资源。

### 7.2 已有能力

- 查看当前全部场景
- 切换当前场景
- 新增场景
- 删除当前选中场景

### 7.3 操作方式

#### 新增场景

操作步骤：

1. 打开 `数据集与方案管理`
2. 点击右上区域的 `新增场景`
3. 输入场景名称和说明
4. 保存后场景加入场景列表

#### 切换场景

操作步骤：

1. 在场景按钮区域点击目标场景
2. 页面自动加载该场景下的资源卡片和方案列表

#### 删除场景

操作步骤：

1. 选中场景
2. 点击右侧 `删除选中的场景`
3. 在确认弹窗中确认删除

删除影响：

- 删除该场景本身
- 删除该场景下全部数据集
- 删除该场景下全部 Prompt
- 删除该场景下全部知识库
- 删除该场景下全部错题集
- 删除该场景下字段映射配置
- 删除该场景下标注方案

### 7.4 注意事项

- 场景删除属于级联删除
- 删除后无法在页面内恢复

## 8. 数据集管理

### 8.1 功能作用

数据集用于导入待标注 Excel 数据，并在标注工作台中展示。

### 8.2 已有能力

- 查看当前场景下的数据集列表
- 导入一个或多个 Excel 数据集
- 删除数据集
- 导出当前数据集全部行

### 8.3 数据集导入

操作步骤：

1. 进入 `数据集与方案管理`
2. 点击 `数据集` 卡片
3. 打开数据集弹窗
4. 点击选择文件
5. 选择一个或多个 Excel 文件
6. 点击 `导入数据集`
7. 等待导入完成
8. 关闭弹窗后页面会刷新数据集卡片数据

### 8.4 导入后的系统行为

- 后端解析 Excel 表头
- 生成数据集记录
- 把每一行写入数据库
- 保存列结构到 `column_schema`
- 如果已配置人工答案列和标注答案列，系统会在导入时自动计算状态和 TP/TN/FP/FN
- 对长字段生成列表预览缓存，减少工作台卡顿

### 8.5 删除数据集

操作步骤：

1. 打开数据集弹窗
2. 在目标数据集上点击删除
3. 进行二次确认
4. 删除后当前场景数据集列表刷新

### 8.6 注意事项

- 数据集导入后前端按页查询，适合大数据量浏览
- 历史已导入但未生成预览缓存的数据，可使用 `卡顿助手` 补齐

## 9. Prompt 管理

### 9.1 功能作用

Prompt 是标注时发送给模型的模板内容。

### 9.2 已有能力

- 查看 Prompt 列表
- 新增 Prompt
- 编辑 Prompt
- 删除 Prompt
- 导出当前场景 Prompt

### 9.3 Prompt 字段

- 名称
- 角色名
- 内容
- 所属场景

### 9.4 操作方式

#### 新增 Prompt

1. 打开 `Prompt` 卡片
2. 点击新增
3. 填写名称、角色名和内容
4. 保存

#### 编辑 Prompt

1. 打开 Prompt 列表
2. 点击目标 Prompt
3. 修改内容后保存

#### 删除 Prompt

1. 打开 Prompt 列表
2. 在左侧卡片区域或操作区点击删除
3. 二次确认后删除

#### 导出 Prompt

页面支持导出当前场景下 Prompt 数据为文件，算法包导出时也会一并导出。

### 9.5 Prompt 占位符规则

系统当前只识别全角占位符：

```text
｛row.列名｝
｛knowledge｝
｛error_sets｝
```

说明：

- `｛row.列名｝`：读取当前行列值
- `｛knowledge｝`：拼接当前方案关联知识库
- `｛error_sets｝`：拼接当前方案关联错题集

普通 JSON 大括号可直接写：

```json
{
  "GPT4_标注": "是",
  "原因": "示例"
}
```

双大括号 `{{xxx}}` 可用于你自己的返回格式提示，系统不会做替换。

## 13. 标注方案管理

### 13.1 功能作用

标注方案定义一套可执行标注逻辑，包括 Prompt 选择方式、初始化方式、标注方法和并发参数。

### 13.2 已有能力

- 查看方案列表
- 创建方案
- 编辑方案
- 删除方案

### 13.3 方案配置项

- 方案名称
- Prompt 多选
- Prompt 初始化方式
- 自定义 Prompt 初始化方法
- 标注方法
- 并发数
- 知识库
- 错题集

### 13.4 Prompt 初始化方式

支持两种：

#### 自动替换占位符

作用：

- 根据 Prompt 中的全角占位符自动替换

适合：

- 模板简单
- 替换逻辑固定
- 只需要把行数据、知识库、错题集按文本拼进去

#### 自定义 Prompt 处理

作用：

- 把 Prompt、知识库、错题集、字段映射、当前行数据都交给开发人员自定义组装
- 自定义初始化入参使用字典，便于按名称直接读取资源

适合：

- 多角色 Prompt 结构复杂
- 需要自定义筛选知识和错题
- 需要拼装为特定消息格式

入参约定：

```python
prompt_contents = {"角色名": Prompt对象}
knowledge = {"知识名称": "知识内容"}
error_sets = {"错题集名称": "错题内容"}
```

原始资源列表保存在 `context["resource_lists"]`，需要资源 id、排序等元数据时读取。

### 13.5 标注方法

系统支持：

- 默认标注方法
- 自定义标注方法

默认标注方法当前规则：

1. 逐个角色 Prompt 调用大模型
2. 每个 Prompt 返回一个字典
3. 字典必须包含标注答案列
4. 所有角色结果都为“是”时，最终答案为“是”
5. 任一角色结果为“否”时，最终答案为“否”

### 13.6 方案创建操作

1. 打开 `标注方案`
2. 点击 `创建标注方案`
3. 输入方案名称
4. 选择 Prompt
5. 选择 Prompt 初始化方式
6. 如为自定义处理，选择自定义初始化方法
7. 选择标注方法
8. 设置并发数
9. 保存

### 13.7 方案编辑

1. 在方案列表点击 `编辑`
2. 修改配置
3. 保存后工作台切换该方案时使用新配置

### 13.8 方案删除

1. 在方案列表点击 `删除`
2. 二次确认
3. 删除后方案不再可选

## 17. 指标区

### 17.1 功能作用

用于展示当前数据集在当前方案下的状态和评估结果。

### 17.2 当前指标分组

#### 数据量

- 总数
- 未标注
- 已标注
- 排队中
- 标注中

#### 混淆矩阵

- TP
- TN
- FP
- FN

#### 评估率

- 算法准确率
- 正确查全率
- 正确查准率
- 错误查准率
- F1 score
- 业务准确率

### 17.3 当前判定规则

系统按人工答案和标注答案的“是/否”关系判断：

- TP：人工是，标注是
- TN：人工否，标注否
- FP：人工否，标注是
- FN：人工是，标注否

### 17.4 悬浮提示

鼠标悬浮在各指标上时，页面会显示含义和计算公式。

## 18. 标注任务执行

### 18.1 支持的标注方式

- 单条标注
- 批量标注
- 全量标注
- 重新标注

### 18.2 单条标注

操作方式：

1. 在表格某行点击操作列按钮
2. 若该行尚未标注，显示 `标注`
3. 若该行已有结果，显示 `重新标注`

### 18.3 批量标注

操作方式：

1. 使用左侧复选框选中多行
2. 点击 `批量标注`

### 18.4 全量标注

操作方式：

1. 点击 `全量标注`
2. 确认执行

系统当前处理逻辑：

- 跳过当前状态为 `排队中` 和 `标注中` 的行
- 其余行都会进入本次任务
- 已有结果的数据支持重新进入全量标注

### 18.5 停止未完成标注

位置：

- 工作台右上 `更多`

作用：

- 终止当前任务中尚未开始执行的排队项
- 已经在运行中的项继续完成

### 18.6 任务状态展示

任务运行时：

- 页面顶部显示一行任务进度条
- 指标区显示 `排队中` 和 `标注中`
- 表格状态列实时刷新

### 18.7 后台持续执行

用户切换页面、切换数据集、切换方案后，后台任务会继续运行。再次切回时，工作台会重新拉取最新状态。

## 20. 表格筛选、排序与搜索

### 20.1 列筛选

每个支持筛选的表头右侧带有漏斗图标。

功能：

- 输入关键词筛选该列
- 支持筛选空值

## 34. 后台扩展点说明

当前系统把需要业务开发人员自己实现的方法集中放在 `user_hooks/` 包中。

### 34.1 目录结构

```text
user_hooks/
├── llm_chat.py
├── annotation_methods.py
├── prompt_init_methods.py
├── analysis_methods.py
└── prompt_utils.py
```

### 34.2 四类扩展点

#### 1. 默认模型调用方法

文件：

- `user_hooks/llm_chat.py`

主要方法：

- `llm_chat_function(prompt) -> dict`
- `llm_dialog_stream_function(payload) -> Iterator[str]`

作用：

- 前者用于默认标注
- 后者用于对话页流式输出

#### 2. 自定义标注方法

文件：

- `user_hooks/annotation_methods.py`

作用：

- 支持一个方案绑定一个自定义标注方法
- 入参为多个已渲染 Prompt 和上下文

#### 3. 自定义 Prompt 初始化方法

文件：

- `user_hooks/prompt_init_methods.py`

作用：

- 在标注前自行拼装每个角色 Prompt
- 入参里 Prompt、知识库、错题集均为字典结构，可通过角色名或资源名称直接读取

返回约定：

```python
{
  "角色名": {
    "prompt_id": "...",
    "name": "...",
    "role_name": "...",
    "content": "..."
  }
}
```

#### 4. 自定义分析方法

文件：

- `user_hooks/analysis_methods.py`

作用：

- 接收当前行数据和标注结果
- 返回分析结果字典

### 34.3 开发人员需要知道的输入信息

在标注和分析过程中，系统会把以下信息传给后端方法：

- 当前行完整数据
- 字段映射配置
- 当前方案信息
- 当前方案关联 Prompt
- 当前方案关联知识库
- 当前方案关联错题集
- 当前行最新标注结果

## 38. 使用建议

- 先完成字段映射，再进入标注工作台，结果判定会更稳定
- Prompt 中统一使用全角占位符，JSON 返回示例直接写普通大括号
- 并发数量以业务模型实际承载能力为准，页面允许配置，后台仍会做任务调度
- 大数据集首次导入后，建议执行一次列表浏览和详情查看确认预览缓存效果
- 老数据迁移后，建议执行一次 `卡顿助手`
